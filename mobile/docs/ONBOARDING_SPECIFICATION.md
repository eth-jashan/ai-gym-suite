## Flow Overview

### Philosophy

The onboarding flow follows a **conversational AI companion** approach where:
- The AI mascot guides users through questions
- Responses feel personalized and adaptive
- Data collection is interspersed with motivational content
- Visual feedback keeps users engaged during a lengthy questionnaire

interface OnboardingState {
  // Progress
  currentStep: number;
  completedSteps: number[];

  // User Data
  name: string;

  // Goals & Motivation
  primaryGoal: PrimaryGoal | null;
  pastObstacle: PastObstacle | null;
  challenges: Challenge[];

  // Experience
  fitnessLevel: FitnessLevel | null;

  // Workout Preferences

  workoutTime: string | null;      // "HH:mm"

  age: number | null;

  currentWeight: number | null;    // kg

  unitSystem: 'metric' | 'imperial';

  // Computed

  setName: (name: string) => void;
  setPrimaryGoal: (goal: PrimaryGoal) => void;
  setPastObstacle: (obstacle: PastObstacle) => void;
  setChallenges: (challenges: Challenge[]) => void;
  setFitnessLevel: (level: FitnessLevel) => void;
  setWorkoutDuration: (duration: number) => void;
  setEquipment: (equipment: Equipment[]) => void;
  setWorkoutTime: (time: string) => void;
  setWorkoutDays: (days: Day[]) => void;

  setCurrentWeight: (weight: number) => void;
  setTargetWeight: (weight: number) => void;

## Accessibility Considerations

1. **Screen Reader Support**
   - All interactive elements have accessibility labels
   - Selection state announced on change
   - Progress indicator announces current step

2. **Motion Sensitivity**
   - Respect `reduceMotion` preference
   - Provide instant transitions as fallback

3. **Touch Targets**
   - Minimum 44pt touch targets
   - Adequate spacing between options

4. **Color Contrast**
   - Text meets WCAG AA standards
   - Selected states have clear visual distinction
